Frequently Asked Questions

1. What type of problem does CCruncher try to solve ?

CCruncher is a program that uses the Monte Carlo method to compute the credit risk of large portfolios of mortgages, loans, bonds, endorsements, or the like (all of them fixed income with a policy buy/sell and hold). Default times are simulated using a copula (gaussian or t-Student), taking into account the transition matrix (or survival functions) and a sectorial correlation matrix.

2. Why this ugly name ?

CCruncher calculates the credit portfolio risk valuation with intensive numerical computation. These kind of programs are called number crunchers. The mix between the terms 'credit' and 'number cruncher' generates the name 'credit cruncher'. CCruncher is not related to the recent credit crunch, but is a tool that can aid to control the risks involved in this crisis.

3. What is the project's maturity ?

ccruncher-1.5 is a stable version. If you have any bug or suggestion to report, please send an email to ccruncher team or post it to SourceForge.

4. Is there documentation where the CCruncher algorithm is explained?

The document that explains the algorithm is the Technical Document. This document is included in all the distributions under the directory docs/. Also you can find a brief introduction of CCruncher in ERCIM News 78.

5. Is the Windows platform suported ?

Yes, but the main stream is developed for the UNIX environment. If you want to compile ccruncher under Windows, read the file docs/README. The repository contains the project files for MSVC9 compiler (*.sln, *.vcproj).

6. Why can the input files be gzipped (*.gz) but not zipped (*.zip) ?

The zip algorithm is covered by patents, but gzip is not; furthermore gzip is open source. You can download a gzip tool for windows from 7-zip. UNIX environments have the gzip/gunzip commands out of the box. You can read more at the gzip website.

7. Is CCruncher fast ?

Yes. It is designed to be very fast. It allows you to evaluate the credit risk of your whole portfolio each day and has been tested extensively with a portfolio of over 50.000 borrowers.
