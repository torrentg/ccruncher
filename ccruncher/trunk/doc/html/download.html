Download

Select a distribution and download it.

Binary release. With this option you don't need to compile nothing. It is ready to use. Source code not included. Only for Linux environment.

Source release. This package contains all elements for create the application. You need to download external libraries (see installation section) and compile the code.

Repository. You can obtain last version from repository. You need a subversion client that you can download from subversion.tigris.org (tortoisesvn is a nice option for Windows users). This option is risky since current repository version can be unstable. Do a svn checkout http://www.generacio.com/svn/repos/ccruncher/trunk/ to download the code from repository.

Installation

Step 1. If you have selected a packed distribution, unpack it.


          tar -xvzf ccruncher-X.Y_ZZZ.tgz
          cd ccruncher-X.Y

Step 2. CreditCruncher use a library named expat. Check that you have the expat package installed (find file libexpat.so, usually in folder /usr/lib). Source distribution requieres that you have the expat-devel package installed (find file expat.h, usually in folder /usr/include). If you use a RPM system, install packages expat and expat-devel.

Step 3. CreditCruncher use a library named zlib. Check that you have the zlib package installed (find file libz.so, usually in folder /usr/lib). Source distribution requires the zlib-devel package installed (find file zlib.h, usually in folder /usr/include). If you use a RPM system, install packages zlib and zlib-devel.

Step 4. CreditCruncher use a statistical package named R to compute risk indicators. Install it from http://www.r-project.org/. If you plan to realize the statistical computations with your own tools (pe. Excel) skip this step. If you use a RPM system, install package R.

Step 5. If you have a source distribution, run next commands (pay attention to $PWD): 


          ./configure --prefix=$PWD
          make
          make install

If you want create a MPI version (to run ccruncher in a cluster) type: 


          ./configure --prefix=$PWD --enable-mpi
          make
          make install

Usage

Step 1. Create a XML ccruncher input file. First, read Technical Document and Input File Reference. You can use a file from directory samples as template. Remain that xml input can be gziped (gz extension) to reduce the input file size. Caution, gzip is a compresion algorithm distinct than zip.

Step 2. Run ccruncher 


          bin/ccruncher --help
          bin/ccruncher -vf --hash=100 --path=./data file.xml

Step 3. Check generated output (plain text files). First field is the simulation counter, second field is the simulated segment value. 


          ls -l data/*
          more data/segmentation-segment.out

Step 4. Compute the Value at Risk 


          bin/report data/segmentation-segment.out

or open a R console and type: 


          > source("$CCRUNCHER/bin/report.R")
          > x <- ccruncher.read("$CCRUNCHER/data/file.out")
          > ccruncher.summary(x, alpha=0.99)
          > ccruncher.plot(x, show="pdf")
          > ccruncher.plot(x, show="cdf")
          > ccruncher.plot(x, alpha=0.95, show="mean")
          > ccruncher.plot(x, alpha=0.95, show="stddev")
          > ccruncher.plot(x, alpha=0.95, var=0.99, show="VaR")
          > ccruncher.plot(x, alpha=0.95, var=0.99, show="TCE")
          > ccruncher.plot(x, alpha=0.95, var=0.99, show="all")
          > quit(save='no')
