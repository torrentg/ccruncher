CreditCruncher - FAQ

1. What is the project maturity ?

CreditCruncher-0.8 have the main framework builded and produce outputs. Now we are on beta developement stage (next stage will be production/stable). We need to translate documentation, check results, find bugs, improve documentation, etc. We call to arms to the net comunity. If you desire to take part in this fase, please send a mail to ccruncher team.

2. Can I found docs where the CreditCruncher algorithm is explained?

Yes. The document is named CreditCruncher - Technical Document and you can found a link here. This document is included in all distributions under the directory docs/. It is awaiting a brave man to translate from spanish to english.

3. Windows platform is suported ?

Yes, but the main stream is in unix environement. If you desire compile the project using Windows, the repository contains the project files for MSVC7 compiler (*.sln, *.vcproj).

4. Why input files can be gziped (*.gz) but not ziped (*.zip) ?

zip algorithm is cover by patents and gzip no, furthermore gzip is open source. You can download a gzip tool for windows from 7-zip. Usually, unix environements have gzip/gunzip commands. You can read more at gzip home.

5. CreditCruncher is fast ?

Yes. CreditCruncher is designed to be very fast. It allows you to evaluate the credit risk of your whole portfolio (can be higher than 50.000 clients) each day. If ccruncher sunk your computer, you can execute it on a cluster enabling the MPI compilation option.
