CreditCruncher - FAQ

1. What is the project maturity ?

CreditCruncher-0.6 have the main framework builded and produce outputs. It is to be about to run out last steps of alpha developement stage. Current open works yet open are:

  • Documentation is under construction (this works slow)

When these works will be closed we will start beta developement stage where non-developers users will test ccruncher. If you desire to take part in this fase, please send a mail to ccruncher team

2. Can I found docs where the CreditCruncher algorithm is explained?

Yes. The document is named CreditCruncher - Technical Document and you can found a link here. This document is included in all distributions under the directory docs/. At this moment, it is under developement and only in spanish. It will be translated to english when completed.

3. Windows platform is suported ?

Yes, but the main stream is in unix environement. The source distribution include the project files for MSVC7 compiler (MSVC6 is not suported).

4. CreditCruncher have a graphic environement ?

No. CreditCruncher has been designed to work in batch mode, without graphical support. The output are plain text files containing the portfolio values obtained in each simulation. In adition, CreditCruncher supply utilities for a report generation (graphics, etc.) using unix utilities (shell scripts, gnuplot). If the user desire make this work, he can take output files and process with Excel to produce the distribution of portfolio value and compute VAR.

5. Why input files can be gziped (*.gz) but not ziped (*.zip) ?

zip algorithm is cover by patents and gzip no, furthermore gzip is open source. You can download a gzip tool for windows from 7-zip. Usually, unix environements have gzip/gunzip commands. You can read more at gzip home.

6. CreditCruncher is fast ?

Yes. CreditCruncher is designed to be very fast. It allows you to evaluate the credit risk of your whole portfolio each day. If ccruncher sunk your computer, you can execute it on a cluster enabling the MPI compilation option.
