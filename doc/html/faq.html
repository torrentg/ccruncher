Frequently Asked Questions

1. What is the project maturity ?

The main goal of CreditCruncher is compute the VAR for a large credit risk portfolio (about 50.000 clients grouped in some sectors) using MonteCarlo.
At this moment (version 0.2 alpha), the main goal is not yet achieved. The reasons are:

  • The portfolio price algorithm need to be reviewed (not critical)
  • The copula generator uses an algorithm based on a matrix NxN. A matrix of 50.000x50.000 elements is not acceptable. Even so, works well for some hundreds of clients.
  • Documentation is under construction (this works slow)
  • The parallel procesing code is not yet implemented

In short, CreditCruncher-0.2 have the main framework builded and produce outputs, but can't be used in a real environement, yet. Is in alpha developement stage.

2. Windows platform is suported ?

At this moment, no. Exist a version compiled with cygwin that shows the capabilities to Windows users. Performance of this executable is very poor (more than 10 times slower) due to cygwin system calls. This project can't be compiled with Microsoft Visual C++ (6) because use embeded projects (MersenneTwister and TNT/Jama) that have problems with this compiler due to an obsolete implementation of C++ standar (has variable scope non ISO C++ standar). Exist an attemp to compile with the NET compiler.

3. CreditCruncher have a graphic environement ?

No. CreditCruncher has been designed to work in batch mode, without graphical support. The output are plain text files (field separator = tab) containing the aggregators value for each simulation. In adition, CreditCruncher will supply utilities for a report generation (graphics, etc.) using unix utilities (shell scripts, gnuplot). If the user desire make this works, he can take output files and process with Excel to produce the distribution of portfolio value at time t and compute VAR.
